# Astral

AI 主持的魔法少女互动小说引擎 · 12 NPC 并发模拟 · 3 个剧本 · 47 个 API

---

## 下载安装（玩家）

### 首次安装

1. 打开 [Releases](https://github.com/Gulu-Lemon/Astral/releases) 页面
2. 下载对应平台的文件：
   - Windows: `Astral-windows-*.zip`
   - macOS: `Astral-macos-*.dmg`
3. Windows 用户解压到任意目录后双击 `Astral.exe`；macOS 用户挂载 dmg 后拖入 Applications
4. 浏览器自动打开 http://127.0.0.1:8640
5. 首次使用会自动弹出 API 设置 → 添加 API 配置

### 从旧版本更新

下载新版 zip/dmg，解压后**将以下文件/文件夹从旧目录复制到新目录**即可保留全部进度：

| 保留 | 内容 |
|------|------|
| `saves/` | 存档文件 |
| `cards/` | 自建角色卡 |
| `config_profiles.json` | API 配置 |

> 其余文件可全部替换。这三个目录/文件是唯一的用户数据，跨版本迁移只需复制它们。

## 快速开始（开发者）

> **要求 Python 3.12+**

```bash
# 首次
git clone https://github.com/Gulu-Lemon/Astral.git
cd Astral
pip install -r requirements.txt
python server.py

# 更新
git pull
pip install -r requirements.txt
python server.py
```

或直接双击 `启动.bat`。

## 局域网其他设备访问

1. 命令行 `ipconfig` 获取本机 IPv4（如 `192.168.1.100`）
2. Win+R → `wf.msc` → 入站规则 → 新建规则，放行端口 8640（TCP）
3. 同一 WiFi 下，其他设备打开 `http://192.168.1.100:8640`

## 角色卡

自建角色或使用预设。在游戏内编辑器或 `cards/` 目录创建 `.txt` 文件：

```
name: 角色名
age: 16
appearance: 外貌描述
magic: 魔法能力描述
personality: 性格描述（可选）
```

## API 配置

配置文件：`config_profiles.json`。支持多配置（DeepSeek / OpenAI / Gemini 等），一键切换。

支持三个组件分别指定不同的模型（在设置面板中展开"高级"即可配置 Agent / Arbiter / GM 各自使用的模型）。

### 模型选择建议

| 组件 | 推荐模型 | 说明 |
|------|---------|------|
| **Agent（12 个 NPC）** | 经济型（如 deepseek-v4-flash、gpt-4o-mini） | 每轮 12 次并发调用，高并发高消耗。**避免按次/按量计费的模型**，优先使用包月或极低价按量模型 |
| **Arbiter（仲裁官）** | 强推理型（如 deepseek-v4-pro、claude-sonnet-4） | 需要逻辑判断、冲突检测和掷骰。建议开启**思考模式（thinking）**提升推理质量 |
| **GM（叙述层）** | 文学性好的模型（如 deepseek-v4-flash、gpt-4o） | 负责叙事生成，对话量大但调用频次低（每轮 1-2 次）。对 token 单价敏感，建议使用包月模型 |

> GPU 算力充足的玩家可以将 GM 也换成强推理模型，叙事质量会显著提升。但 Agent 最重要的指标是**低价**而非质量——12 个 Agent 每轮同时请求，按量计费模型会迅速消耗余额。

## 存档

自动存档 + 手动槽位。存档文件在 `saves/` 目录。

## Debug

运行后 `logs/` 目录自动生成环形缓冲日志（每文件最多 5000 行）：

| 文件 | 内容 |
|------|------|
| `requests.log` | API 请求耗时 |
| `agents.log` | Agent 决策记录 |
| `errors.log` | 全局异常堆栈 |
| `thread_errors.log` | 线程异常 |

## 技术栈

- 后端：Flask + SSE 流式推送 + ThreadPoolExecutor
- 前端：Vanilla HTML/CSS/JS + EventSource
- LLM：OpenAI 兼容 API（config_profiles.json 管理）
- 存档：JSON
