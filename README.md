# Astral v0.5

多 Agent 互动小说引擎 · 12 NPC 并发模拟 · 3 个剧本 · 37 个 API · 角色卡

> 详细功能清单见 [FEATURES.md](FEATURES.md)

---

## 快速开始

双击 `启动.bat` → 浏览器自动打开 http://127.0.0.1:8640

首次使用：
1. 页面自动弹出 API 设置 → 添加并选择一个 API 配置 → 自动测试连通性
2. 选择场景（天际迷宫 / 云端假期 / 风雪列车）
3. 选择已有角色卡，或点击"+ 新建角色卡"填写名字/外貌/魔法/性格
4. 选择难度模式 → 序章自动推进
5. 进入主循环

---

## 局域网其他设备访问

手机 / 平板 / 其他电脑均可通过浏览器访问本机：

**1. 获取本机 IP**

命令行运行 `ipconfig`，找到 IPv4 地址，如 `192.168.1.100`。

**2. 防火墙放行端口 8640**

Win+R → `wf.msc` → 入站规则 → 新建规则：
- 规则类型：端口
- 协议：TCP，特定本地端口 `8640`
- 操作：允许连接
- 配置文件：全部勾选
- 名称：Astral

**3. 访问**

同一 WiFi 下，其他设备打开浏览器访问 `http://192.168.1.100:8640`。

> 无需任何代码改动。Flask 绑定 `0.0.0.0`，已监听全部网络接口。手机端 UI 在窄屏下可正常使用（滚动 + 缩放）。

## 三阶段游戏进程

| 阶段 | 触发条件 | Agent 权限 |
|------|----------|------------|
| **黑箱沉默期** | 游戏开始 | 社交/探索，禁止攻击和破坏 |
| **暗流缓冲期** | 认识全部 NPC | 开放跟踪/孤立/对峙 |
| **猎杀期** | 首起案件后 | 开放攻击(需无人目击)和陷阱 |

---

## 攻击与陷阱

| 行动 | 条件 | 说明 |
|------|------|------|
| **ATTACK（直接攻击）** | 同房间无其他 NPC | 有目击者自动降级为对峙 |
| **TRAP（陷阱）** | 任何场景 | 风险 = LLM 自评 + 旁观者人数加成 |
| 尸体发现 | 需 2+ 人同位置 | 触发魔女审判（搜查→陈述→辩论→论告→投票→处刑） |
| 毁尸灭迹 | 不允许 | 道德底线或意外失败，尸体和线索保持存在 |

## 好感度等级

0-9 仇恨 → 10-19 敌对 → 20-29 冷淡 → 30-39 陌生 → 40-49 相识 → 50-59 友人 → 60-79 知己 → 80-100 恋人

## 风险判定（d6 随机）

| 风险等级 | 成功概率 |
|----------|----------|
| 不可能 | 0% |
| 极高风险 | 1/6（骰出 0） |
| 高风险 | 2/6（骰出 0-1） |
| 中风险 | 3/6（骰出 0-2） |
| 较低风险 | 4/6（骰出 0-3） |
| 低风险 | 5/6（骰出 0-4） |
| 无风险 | 100% |

## Debug

运行后 `logs/` 目录自动生成：

| 文件 | 内容 | 用途 |
|------|------|------|
| requests.log | 所有 API 请求耗时 | 排查按钮无响应 |
| agents.log | 12 Agent 每轮决策记录 | 排查 NPC 行为异常 |
| errors.log | 全局异常堆栈 | 排查崩溃/SSE 中断 |
| thread_errors.log | 线程未捕获异常 | 排查并发错误 |

每个文件最多 5000 行，自动覆盖。

## 技术栈

- **后端**: Flask, SSE 流式推送, ThreadPoolExecutor(8 workers)
- **前端**: Vanilla HTML/CSS/JS, EventSource
- **LLM**: OpenAI 兼容 API（config_profiles.json 多配置管理）
- **存档**: JSON, saves/ 目录

## 项目结构

```

├── server.py             # Flask 服务端（37 个 API 端点）

├── card_manager.py       # 角色卡文件管理
├── config_profiles.py    # API 多配置管理
├── save_manager.py       # 存档系统（自动 + 6 手动槽位）
├── debug.py              # 日志系统（4 个环形缓冲日志文件）
├── scenarios/            # 场景包（3 个内置）
├── static/               # 前端（index.html, style.css, app.js）
├── cards/                # 角色卡文件存放目录
├── saves/                # 存档目录
├── logs/                 # 调试日志
└── config_profiles.json  # API 配置（多配置）
```
