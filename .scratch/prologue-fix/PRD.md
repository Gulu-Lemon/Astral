# Prologue 胡编乱造修复

## 背景

序章存在严重的 LLM 虚构问题：NPC 行为偏离档案、事件凭空编造、故事随轮次累积放大的自循环错误。

## 根因链

```
全量NPC档案注入（含秘密/动机）
  → LLM 即兴填补叙事空白 → 编造事件
  → _extract_beat() 抓取第一句 → 写入故事节拍
  → 下轮 prompt 把编造事实当作【故事脉络】
  → 滚雪球放大
  → 上下文截断替换详细历史
  → prologue_finish() 压缩为单条摘要 → 正文继承假故事
```

## 目标

- 消除故事节拍自循环
- 限制 NPC 信息暴露（按玩家已知过滤）
- 强化约束与校验
- 建立序章→正文的可靠桥接
