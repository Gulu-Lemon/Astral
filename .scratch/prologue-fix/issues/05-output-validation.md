# P1-05 — 序章 LLM 输出校验

Status: done

## 实现

在 `_safe_llm()` 中自动触发（`prologue_step < 7` 时），零改动调用方：

1. **真名泄露检测**：扫描输出中是否出现 `player_met_npcs` 之外的 NPC 真名
2. **异常叙事检测**：搜索暴力/死亡/超自然关键词
3. 设施一致性跳过（关键词匹配不足以判断，需要语义理解）

非阻塞：仅 `_log("validate", ...)` 警告，不打断用户体验。

## 问题

序章唯一的"校验"是正则匹配 `【选项】A. ... B. ...` 格式。**完全没有校验：**
- 是否使用了 NPC 真名
- 是否编造了事件
- 是否尊重了房间设施
- 是否泄露了秘密
- 叙事质量是否合理

## 修复方案

在 `_safe_llm()` 返回后、发送前增加**轻量校验层**：

1. **真名泄露检测：** 扫描 LLM 输出中是否出现 NPC 名字（与 `player_met_npcs` 比较，未见面 NPC 的名字出现则告警）
2. **房间设施一致性：** 提取 LLM 输出中的物品/地点名词，与 `room_features` 比较
3. **异常叙事检测：** 检测暴力/死亡/超自然关键词，若序章中出现则标记

校验失败时：
- **非阻塞：** 记录日志警告，仍返回给玩家（避免打断体验）
- **可配置：** 紧要场景可选择阻塞并重试

## 影响文件

- `server.py`：`_safe_llm()` 后增加 `_validate_prologue_output()`
- `debug.py`：新增 prologue 校验日志类别
